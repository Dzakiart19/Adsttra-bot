## What's committed to GitHub (must be present for clone to work)
- `src/` — all TypeScript source
- `.replit` — env vars, build command, run command, workflows
- `replit.nix` — Nix packages (mesa, freetype, libX*, etc.)
- `package.json` — dependencies + postinstall stub
- `package-lock.json` — **committed** (removed from .gitignore 2026-07-23); ensures exact dep versions
- `.puppeteerrc.cjs` — redirects puppeteer cache to `.puppeteer_cache/` (project-local, persists in deployment)
- `start.sh` — dynamic startup script
- `useragent/` — UA JSON files

## What's gitignored (regenerated automatically)
| File/Dir | Regenerated by |
|---|---|
| `node_modules/` | `npm install` (build) |
| `dist/` | `npm run build` (build) |
| `.puppeteer_cache/chrome-headless-shell/` | `npx puppeteer browsers install` (build) |
| `.nix_env` | `printf REPLIT_LD_LIBRARY_PATH > .nix_env` (build) |
| `proxy_cache.json` | ProxyService at runtime |
| `uptime_stats.json` | UptimeService at runtime |
| `.cache/` | Replit/Nix at runtime |

## Build command flow (`.replit [deployment] build`)
```bash
npm install
npm run build
rm -f proxy_cache.json uptime_stats.json
rm -rf .puppeteer_cache/chrome-headless-shell
PUPPETEER_CACHE_DIR=.puppeteer_cache npx puppeteer browsers install chrome-headless-shell
printf '%s|' "${REPLIT_LD_LIBRARY_PATH}" > .nix_env
```
All steps are idempotent and work from a clean state.

## npm postinstall
Set to `echo 'Skipping browser download'` — intentionally prevents puppeteer from downloading
Chrome during `npm install`. Chrome is downloaded separately by the build command into `.puppeteer_cache/`.

## Environment variables
All required env vars are in `.replit [userenv.shared]` — they travel with the repo and are
automatically applied in both dev and production. No manual setup needed.
`SESSION_SECRET` is a Replit Secret but is NOT used in the codebase — safe to ignore.

## Deployment type
`autoscale` — user is on free Replit tier. Bot stays alive via `/health` cronjob pinging the endpoint.
The `/health` endpoint (`GET /health`) returns JSON with process status + success rate.

## Why package-lock.json must be committed
Without it, `npm install` resolves latest semver-compatible versions → risk of silent breaking
changes from dependencies between clones. Committing the lockfile ensures identical installs.

## Fresh clone → publish checklist (all automated, no manual steps)
1. Clone repo into new Replit project
2. Replit reads `replit.nix` → sets up Nix environment → `REPLIT_LD_LIBRARY_PATH` available
3. Click Publish → build command runs → Chrome installed, `.nix_env` captured, `dist/` built
4. Run command: `bash start.sh` → reads `.nix_env` → sets `LD_LIBRARY_PATH` → Chrome launches ✅
