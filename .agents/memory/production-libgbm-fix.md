---
name: Production libgbm fix
description: Root cause and fix for "libgbm.so.1: cannot open shared object file" / Chrome exit 127 in autoscale production. Fixed 2026-07-23.
---

## Symptom
Production (autoscale): `Failed to launch the browser process: Code: 127`
```
chrome-headless-shell: error while loading shared libraries: libgbm.so.1: cannot open shared object file: No such file or directory
```
Dev (pike.replit.dev): works fine.

## Root cause
In autoscale production runtime containers, `REPLIT_LD_LIBRARY_PATH` is NOT set as a shell
environment variable. The old build command used a Python script to read
`.cache/replit/nix/env.json`, but that file does not exist in Replit's production build
containers → `.nix_env` was saved with empty values → `LD_LIBRARY_PATH` never set at
runtime → Chrome cannot find `libgbm.so.1`.

## Why dev worked
Dev environment always has `REPLIT_LD_LIBRARY_PATH` set (Nix shell sets it automatically).
start.sh reads it directly → `LD_LIBRARY_PATH` set correctly → Chrome finds libgbm.so.1.

## Fix applied (2026-07-23)
Changed `.replit` build command from Python+env.json to:
```bash
printf '%s|' "${REPLIT_LD_LIBRARY_PATH}" > .nix_env
```
`REPLIT_LD_LIBRARY_PATH` IS available during the BUILD step (Nix sets it before any command
runs, even in production build containers). Capturing it at build time → saved in `.nix_env`
→ start.sh reads `.nix_env` at runtime → `LD_LIBRARY_PATH` set correctly → Chrome works.

## Key insight
`REPLIT_LD_LIBRARY_PATH` is available at BUILD time but NOT at autoscale RUNTIME.
Solution: capture at build, persist to `.nix_env`, read at runtime.

## How to apply
If Chrome fails with Code 127 / libgbm missing in production:
1. Verify build command uses `printf '%s|' "${REPLIT_LD_LIBRARY_PATH}" > .nix_env`
2. Verify `.nix_env` is NOT gitignored (it must be regenerated by build, but not tracked)
3. Verify start.sh reads `.nix_env` as fallback when `REPLIT_LD_LIBRARY_PATH` is empty
4. Republish to trigger fresh build
